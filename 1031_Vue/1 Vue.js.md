## 11/1 Vue.js



- Vue Style Guide
  
  - v-for는 항상 key와 함께 사용하기
    
    - 데이터의 예측 가능한 행동을 유지시키기 (객체 불변성)
    
    - 내부 컴포넌트의 상태를 일관되게 유지하기 위해
  
  - v-for를 쓴 엘리먼트에 절대 v-if를 사용하지 말기
    
    - vue가 디렉티브를 처리할 때, v-if가 v-for보다 높은 우선순위를 가짐
    
    - 서로 다른 태그로 감싸기
    
    - 함께 쓸수 있다고 생각되는 2가지 경우
      
      1. 목록의 항목을 필터링할때
      
      2. 숨김 목록의 렌더링을 피할 때
